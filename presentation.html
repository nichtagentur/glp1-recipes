Zero human intervention

Once deployed, the system runs autonomously. The topic pool of 507 recipes ensures over 100 days of unique content. Category rotation keeps the mix varied. A generated-log.json prevents duplicates.

Graceful error handling

If Claude fails on one recipe, the batch continues with the remaining 4. If Gemini fails on an image, the recipe still publishes without a photo. No single failure stops the pipeline.

Content Generation

Claude writes the recipes.

// System prompt enforces nutritional targets const SYSTEM_PROMPT = `You are a professional nutritionist and recipe developer... Meals: 300-500 calories, 20-40g protein Snacks: 100-200 calories, 8-15g protein All recipes: high fiber (5g+), easy to digest Respond ONLY with valid JSON.` // Call Claude Sonnet 4.6 const message = await client.messages.create({ model: 'claude-sonnet-4-6', max_tokens: 2000, system: SYSTEM_PROMPT, messages: [{ role: 'user', content: prompt }] }); // Returns structured JSON with all fields const recipe = JSON.parse(message.content);

Output per recipe:

  • Title + description -- SEO optimized
  • Full nutrition -- cal, protein, carbs, fat, fiber
  • Ingredients -- precise measurements
  • Instructions -- clear numbered steps
  • Tags -- cuisine, diet, key ingredients
  • Image prompt -- for Gemini photo generation
  • Intro paragraph -- body text for the page

SEO Implementation

Built for Google from day one.

Schema Markup (JSON-LD)

  • Recipe schema -- name, image, author, times, nutrition, ingredients, instructions
  • BreadcrumbList -- Home → Category → Recipe
  • Nutrition info -- calories, protein, carbs, fat, fiber per serving

Meta Tags

  • Title tags -- 50-60 chars, keyword optimized
  • Open Graph -- og:title, og:image (1200x800), og:url
  • Twitter Card -- summary_large_image
  • Canonical URLs -- on every page

E-E-A-T / YMYL Compliance

  • About page -- editorial mission, process, sources
  • Disclaimer -- on every page footer + dedicated page
  • No medical claims -- framed as nutrition info
  • Authority links -- NIH, FDA, NIDDK
  • Author attribution -- consistent across all content

Site Structure

  • 8 categories -- breakfast through side dishes
  • Auto sitemap -- via @astrojs/sitemap
  • Zod validation -- catches malformed content at build
07 / 10
GitHub Actions

Two workflows, full automation.

Workflow 1
generate.yml
Daily cron at 6:15 AM UTC + manual dispatch
on: schedule: - cron: '15 6 * * *' workflow_dispatch: steps: - Checkout repo - Setup Node 22 - npm ci - node scripts/generate-batch.mjs # env: CLAUDE_API_KEY, GEMINI_API_KEY - git add + commit + push
Workflow 2
deploy.yml
Triggered on every push to main
on: push: branches: [main] steps: - Checkout repo - withastro/action@v3 # Builds the Astro site - actions/deploy-pages@v4 # Deploys to GitHub Pages

Chain reaction: generate.yml pushes new content → triggers deploy.yml → site rebuilds → new recipes live within minutes.

08 / 10
Quality Control

Multiple safety nets.

Zod
Content collection schema validates every recipe at build time. Missing fields, wrong types, or out-of-range nutrition values break the build before publishing.
System Prompt
Claude's system prompt enforces strict nutritional targets. Meals must hit 300-500 cal, 20-40g protein. The AI cannot deviate from the guidelines.
Log
generated-log.json tracks every recipe ever created. Prevents duplicates across runs. Enables category rotation for maximum topic variety.
